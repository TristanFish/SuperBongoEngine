460  /*** Note 2.
461  In straight C programming, lvalues and rvalues literially means left and right assignments. For example,
462  int x = 5; x is the lvalue and 5 is the rvalue. Easy. In C++ it get a bit more tricky. The modern idea is
463  rvalues are temporary vaules residing in the registers of the CPU. lvalues are actual memory locations.
464  In the code:
465  inline float& operator [] ( int index ) {
466  return *(&x + index);
467  }
468  To read this precicely, &x is the address of the x variable (the first in the list of x,y,z) add to that
469  address the index as if it were an array. Then *(&x + index) dereferences that address back into the
470  object, the float& reference returns it as an address and thus an lvalue.
471  ***/
